Giriş:

🔹 Monolithic ve Microservice mimarilerinin farklarını
🔹 API Gateway yaklaşımını
🔹 RabbitMQ ve MassTransit ile servisler arası asenkron iletişimi
🔹 Event Driven Architecture yaklaşımını
🔹 Docker ve Docker Compose kullanımını
🔹 Redis, MongoDB, PostgreSQL ve SQL Server gibi farklı veri tabanlarının kullanım senaryolarını
🔹 JWT, Identity Server ve token yönetimini
🔹 DDD (Domain Driven Design), Repository Pattern ve Unit Of Work yaklaşımlarını
🔹 Eventual Consistency ve dağıtık sistemlerde veri tutarlılığını
uygulamalı olarak deneyimleme fırsatı buldum.

Kullanılan Teknolojiler Ve Uygulamalar:

 ->Asp.net Core Api 6.0
 ->Asp.net Core Mvc (api istekleri ve web arayüzü için)
 ->Onion Architecture
 ->DB:MS SQL Server, MongoDB, PostgreSQL
 ->Microservices Architecture(Beginner)
 ->Entity Framework (Code First),Dapper
 ->Identity Server4, Token exchange ile token değişimi
 ->Fluent Validation
 ->Repository Design Pattern
 ->CQRS Design Pattern
 ->Mediator Design Pattern
 ->Unit Of Work Design Pattern
 ->Dependency Injection

Özellikle mikroservislerde her servisin kendi verisine sahip olması, servisler arası bağımlılıkların azaltılması, Eventual Consistency yaklaşımı ve mesajlaşma sistemleriyle veri tutarlılığının nasıl sağlandığını görmek benim için oldukça değerliydi.
Bu süreçte sadece çalışan bir proje geliştirmedim; aynı zamanda büyük ölçekli sistemlerin nasıl tasarlandığını ve neden bu şekilde tasarlandığını anlamaya çalıştım.
